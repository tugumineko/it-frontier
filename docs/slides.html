代码词法地图

编译器词法分类 × LLM 逐词语义，帮编程初学者看懂每个单词

出发点

初学者面对代码的第一个困惑

现状

AI 代码解释工具（ChatGPT、Copilot Chat）以整行或整段为粒度给出笼统说明，跳过了最基础的一步。

语法高亮的局限

IDE 能给不同类型的词着色，但无法告诉初学者「这个词在这段代码里具体干什么」。

我们要解决的问题

以 token（词法单元）为粒度，逐个告诉初学者：这个词属于哪类，在当前上下文中起什么作用。

核心创新

编译器词法 + LLM 语义的组合创新

将确定性的词法分类和灵活的上下文语义解释在 token（词法单元）粒度上组合。这个组合在已有工具中没有先例。

工具演示

四步体验

  1. 粘贴代码，点击分析
    Pygments 词法分析在 0.1 秒内把代码切成 token（词法单元），分到关键字、标识符、运算符、字面量四区，生成 2D 词法地图。
  2. 点击任意词，看它在这里干嘛
    三视图联动：地图圆点亮起、代码行高亮、语义卡片弹出。固定字典和上下文提示秒出，LLM 深层解释约 1 秒到达后替换。
  3. 分级样例引导
    五个内置样例从「x = 10」到一词多义，降低「不知道从哪下手」的门槛。
  4. 导出与复用
    分析结果可导出 JSON，教师预分析后分发给学生。

技术要点

分级解释策略

第一级 · 固定字典

0 ms。关键字（def、return、if）、运算符（=、+、==）、常见标识符（self、cls）含义恒定，硬编码查表。

第二级 · 上下文提示

0 ms。def 后面 → 「函数名」，class 后面 → 「类名」，import 后面 → 「模块」。先行显示，等 LLM 到了再替换。

第三级 · LLM 深层解释

~1 s。ecnu-max 根据完整代码上下文，给出不超过 20 字的一句话解释。多个请求并发执行。

典型代码中各级覆盖比例

字典 ~35% 上下文 ~25% LLM ~40%

约 60% 的词法单元无需 LLM 即可获得有意义的解释

技术要点

性能优化

从最初的 50 秒到现在的亚秒级响应，经过三轮优化。

v1 一次性 LLM 全解释
50.1 s
v2 词法秒出 + 按需解释
0.6 s + 点击 ~1 s
v3 分级策略 + 多线程并发
0.1 s + 字典秒出 / 并发 0.7 s
优化手段效果
词法分析与语义解释分离地图 0.1 秒出现
固定字典 + 上下文推断60% 的词法单元 0 ms 响应
Flask threaded=True5 个并发请求 3.2 s → 0.7 s
prompt 压缩 + max_tokens 限制单次 LLM 响应更快
reqSeq 序列号 + busy 防重入消除画面切换和请求积压
总结

编译器告诉你「这是什么词」
LLM 告诉你「它在这里干嘛」

两者各司其职，在 token（词法单元）粒度上组合，把词法分析从黑箱变成可交互的初学者教学体验。

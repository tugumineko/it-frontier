# 词义星海：项目说明

> 这份文档讲项目怎么设计、为什么这么设计、怎么跑起来，面向看仓库代码的人。叙事和动机见《讲解文档》，基础概念见《概念入门》。

---

## 一、目标与约束

这一节先说清楚项目在什么约束下做，后面的设计都从这几条来。

- 离线无付费接口：现场断网也要能跑，不依赖任何云端 API。
- 演示稳定：汇报当天不做可能崩的重计算。
- 现场可交互：给任意一句话，要能在几秒内出结果。
- 视觉为主：用图形学把一个抽象的研究对象讲生动。

## 二、关键选型与理由

这一节把几个主要决定和它们的理由摆出来，这是项目的核心。

### 2.1 看「一词多义」这个现象

可解释性的研究对象抽象，需要一个具体好懂的切入口。一词多义问题清楚、有看点、且天生是动态的（同一个词随上下文移动），正好适合用一座可飞行的 3D 星海来呈现。

### 2.2 用上下文向量，并选第 9 层

模型输入层的静态词向量里一个词只有一个坐标，表现不了一词多义。把整句话喂进 GPT-2、取中间层的上下文向量才能体现。取哪一层是先验证后定的：`pipeline/diagnose_polysemy.py` 逐层测义项可分度，第 9 层判别准确率 0.90，明显高于静态层的 0.69 和顶层的 0.82，所以选第 9 层。

### 2.3 固定语义星海 + 探针叠加

星海底图是一个固定的语义参照系，输入句子时把词作为探针点叠加进去，而不是每次重建一套星海。只有参照系固定，「同一个词在不同句子里落到不同区域」这种对比才有意义。

### 2.4 模型选 GPT-2

GPT-2 小、开源、可复现，768 维、12 层，本地几秒能跑完一句话，不需要付费接口。这个项目要说的事跟模型大小无关，小模型反而干净好讲。

### 2.5 计算分两段：构建期算底图，演示期只渲染加投影

底图的词向量、降维、聚类都在构建期用 Python 算好，写进一份 JSON 和一个投影器文件。演示期浏览器只加载和渲染底图；输入句子时由本地后端用同一个投影器现跑取上下文向量。这样底图永远稳，实时部分失败也不影响星海一直亮着。

### 2.6 定位演进

项目早期叫「UMAP 测谎仪」，想证明「UMAP 在撒谎」；真实数据自测后发现这个主张站不稳，于是先收为「语义漫游」，又因为漫游本身没有一个具体问题而显得空。最后落到「一词多义观测」：有具体问题、有看点、且恰好用得上 3D 和实时这套框架。降维在这一版只是把高维画出来的手段，不再是主题。

## 三、架构：构建期与演示期分离

这一节给出数据从哪来、到哪去的整体流向。

```
构建期 (Python, 一次)                  演示期 (浏览器, 离线零接口)
GPT-2 第 9 层上下文向量(模板句)          加载 galaxy.json
  → PCA / UMAP → 3D 星海底图            → Three.js 渲染语义星海
  → KMeans 上色 + 最近邻索引            → 输入一句话
  → 多义词案例义项落点                  → 后端 /api/locate 取每词上下文向量
  → galaxy.json + projector.joblib       → 投影成探针星 + 句子轨迹 + 最近邻
```

构建期算好的底图落在 `galaxy.json`，投影器和最近邻索引落在 `projector.joblib`。实时这条链路由 `server/app.py` 的 `/api/locate` 承担：句子里的每个词取第 9 层向量，用同一投影器投到星海，并查最近的底图词。没装 ML 依赖时后端进 mock 模式，把词散布占位，保交互链路能跑。

## 四、怎么跑

这一节给几种跑法，按依赖从少到多排列。

### 看星海（带实时与案例，推荐）

```bash
pip install -r server/requirements.txt   # 需要 pipeline 的 ML 依赖才有 real 模式
python server/app.py                      # 浏览器开 http://127.0.0.1:5000
```

从 5000 端口打开，输入句子、选多义词案例都可用。

### 重建底图数据

```bash
pip install -r pipeline/requirements.txt
python pipeline/export_semantic_galaxy.py --topk 5000 --layer 9
# 生成 web/data/galaxy.json + projector.joblib
```

### 验证方向（选层依据）

```bash
python pipeline/diagnose_polysemy.py      # 打印各层的义项判别准确率
```

## 五、目录结构

这一节给出仓库布局，方便定位代码。

```
latent-galaxy/
├─ web/                          # 演示前端（纯静态，离线）
│  ├─ index.html
│  ├─ css/style.css
│  ├─ js/
│  │  ├─ main.js                 # 场景 / 相机 / 交互 / 探针层
│  │  ├─ galaxy.js               # 星海渲染 + 探针点与句子轨迹
│  │  ├─ ui.js                   # 面板：输入句子 / 多义词案例 / 展示参数
│  │  └─ data.js                 # 数据加载与归一化
│  ├─ vendor/three/              # 本地 vendored Three.js（保证离线）
│  └─ data/galaxy.json           # 语义星海底图（构建期生成）
├─ pipeline/                     # 构建期（Python）
│  ├─ export_semantic_galaxy.py  # 第 9 层上下文向量 → 星海底图 + 投影器 + 案例
│  ├─ diagnose_polysemy.py       # 逐层测义项可分度（选层依据）
│  └─ requirements.txt
├─ server/
│  └─ app.py                     # 托管前端 + /api/locate（real / mock 自动切换）
└─ docs/                         # 讲解文档 / 项目说明 / 概念入门
```

## 六、数据格式

这一节给出底图 `galaxy.json` 的结构。

```jsonc
{
  "meta": {
    "source": "gpt2:layer9",
    "count": 5000,
    "layer": 9,
    "clusters": [{ "id": 0, "name": "区名", "color": [r, g, b] }],
    "cases": [{
      "word": "bank",
      "senses": [
        { "name": "金融 finance", "xyz": [x, y, z], "uvw": [x, y, z], "example": "..." },
        { "name": "河岸 river",   "xyz": [x, y, z], "uvw": [x, y, z], "example": "..." }
      ]
    }]
  },
  "tokens":  ["money", "river"],   // 长度 N
  "cluster": [0, 1],               // 每点聚类 id（仅上色）
  "pca":     [[x, y, z]],          // PCA 布局
  "umap":    [[x, y, z]],          // UMAP 布局（默认显示，义项分得更开）
  "size":    [0.0]                 // 点大小（词频权重）
}
```

后端 `/api/locate` 返回每个词的 `{word, pca, umap, neighbors}`，前端据此渲染探针点、句子轨迹和最近邻信息卡。

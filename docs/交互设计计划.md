# 交互改进计划（已落地）

> 目标：不破坏沉浸式自由观察的前提下，让 **悬停预览 / 选择 / 聚焦** 各司其职——低惊讶、可发现、可撤销、不眩晕。
> 依据：Norman（可发现性/反馈/低惊讶）、Shneiderman 八大黄金法则（信息反馈/轻松撤销/内部控制点）。

## 1. 原 bug 根因
监听浏览器原生 `click`——它只看 down→up "位移小、间隔短"，**无法表达用户是否在拖拽**。OrbitControls 旋转就是 `down+大位移+up`，松手合成 `click` → 被误判为"选择/聚焦"，相机被强行甩走。违反"内部控制点（用户而非系统发起动作）"。

## 2. 修法（已实现）
- **不监听 `click`**：用 `pointerdown/pointerup` 的「位移 + 时间」阈值自己合成"真单击"：鼠标位移 ≤ **5px**（触摸 10px）且时长 ≤ **300ms**，否则视为拖拽，不拾取。`Math.hypot` 比欧氏距离。
- **`setPointerCapture`**：防 canvas 外松手丢 up。
- **OrbitControls `start/end` 闸门**：拖拽中 `isOrbiting=true` → 不拾取、不弹卡；`end` 后延迟 1 帧复位。

## 3. 交互分层（代价与意图强度对齐，状态优先级 focused > selected > hovered）
| 档 | 触发 | 行为 | 动相机 | 可撤销 |
|---|---|---|---|---|
| 轻·预览 | hover（rAF 节流） | 词卡跟随光标 + 光标变 pointer | 否 | 移开即消 |
| 中·选择 | 真单击 | 持久高亮同簇 + 选中词卡（含飞入/取消按钮）+ 暂停自转 | **否（关键）** | 点空白 / ✕ / Esc |
| 重·聚焦 | 双击 / 词卡"飞入"按钮 / 案例库 | 相机平滑飞入 | 是 | Esc 飞回全景 |

**单击绝不动相机**——即便阈值边界误判，最坏只是选中一个词（点空白即取消），相机永不被甩走。

## 4. 相机飞行不眩晕
- 同步插值 `camera.position` + `controls.target`（只插 position 会松手跳变）。
- **650ms easeInOutCubic**；飞行期 `controls.enabled=false`，落地 `update()` 交还控制。
- `camera.up` 全程 (0,1,0) 不滚转；沿当前视线方向退距，保留方位角不甩头。
- **可中断**：飞行中 `pointerdown` 即 `cancelFlight()` 接管。
- 初始化 `homeState=captureState()` 锚定全景。

## 5. 可发现性与反馈
- 光标：命中=pointer / 空场=grab / 拖拽中=grabbing。
- hover 高亮本帧出现；选中持久高亮 + 词卡。
- 顶部模式条：**自由观察 / 聚焦：<词>**。
- 底部操作提示条：`拖拽=旋转 · 滚轮=缩放 · 右键=平移 · 单击=看词 · 双击=飞入 · ESC=复位`，首次 6s 后淡出，左下 `?` 再唤出。
- **ESC 分级**：先取消选择；再按一次平滑飞回全景。

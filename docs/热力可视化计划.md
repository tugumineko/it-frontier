# 失真热力可视化 · 迭代计划（调研整理）

> 来源：开源数据可视化技法调研（Turbo/viridis 感知均匀配色、降维失真可视化、three.js 拾取、可视化美学）。
> 配色 GLSL 为 Google Turbo(Apache-2.0) / BIDS viridis(CC0) 的公开数值拟合，可直接用。

## 排序原则
先让**星系立住**（结构清晰、背景退后）→ 一维**失真热力**（信息密度最高、低风险）→ **连线**→ **交互**→ 最后才上 bloom/KDE/CheckViz 这类高风险炫技。每层可开关 + 可 lerp，不硬切。

## 1) 展示星系优先（Focus+Context + 亮度预算）
- 背景色=雾色 `0x05060a`，FogExp2 远点沉入（雾色必须=背景色）；vignette 锁中心（已做）。
- **亮度预算铁律**：数据点全饱和最高对比；背景装饰点 alpha ≤ 0.10~0.18。
- **高失真热点对雾/去饱和豁免**——否则远处"测谎红星"被雾吃成灰=误读（最隐蔽的坑）。
- 自定义 ShaderMaterial 不自动吃 fog，需手写。

## 2) 失真热力的正确表达
- **顺序型配色**（失真 0=诚实→1=被骗，无中点）→ 默认 Turbo（沉浸醒目）/ inferno（严谨、暗背景发光感）。**禁用 jet/rainbow/hsv**（亮度非单调、造假边界、色盲不可读）。
- **归一化用稳健分位 2%/98% 裁剪**，绝不 min/max（少数极端词会把全图挤暗）。
- 同一 `d` 同时驱动 **颜色 + 大小 + 亮度**：`size=mix(2,9,d)`、`color=Turbo(d)`、`brightness=(0.6+d*1.4)`，高失真词=报警的红巨星。
- 材质：AdditiveBlending + depthWrite:false + 暗背景；基底亮度 0.15 保证低失真点可见；gl_PointSize 上限 ~63px。

## 3) 近邻连线"测谎"更清晰
- `ratio = d_UMAP / d_PCA`（拉伸比），robust 分位(5/95)归一化 → 上色：ratio≈1 青白、ratio≥4 亮红。
- 透明度/亮度随 ratio：小失真 alpha 0.05 当"银河丝"，大失真 alpha 0.9 高亮。
- **默认只显 ratio_norm>0.6 的高失真边**（滑块控阈值），否则糊成毛线团。
- 单 LineSegments 单 draw call；边色(青/洋红) 与 点色(Turbo) 分离不撞色。

## 4) 交互与自定义
- **hover tooltip**：拾取（<几万点用 raycaster，本项目用屏幕投影最近点更稳，因 GPU morph）+ 全局单 DOM 标签，显示 token + 失真% + 被扯长 Nx。标签 `pointer-events:none`。
- **点击聚焦 + Focus+Context**：相机 `lerp(target,0.08)`；per-point `aFocus` 每帧 lerp 软压暗其余（绝不 visible=false 硬隐藏），收敛后停更。
- **自定义滑块**：背景星亮度 / 点大小 / 连线阈值 / k / 配色(Turbo·inferno·cluster) / Bloom / 雾密度。
- 进阶：Shepard-lens 鼠标透镜（只放大 size/alpha 不动坐标）。

## 5) 图例与研究图
- **配色图例条**（必做）：Canvas2D 同函数 256px 渐变 + 刻度 + 钉死文案"红/亮黄=UMAP 骗得越狠"。
- **失真直方图** HUD：x=失真分 y=词数，柱用同 colormap；hover 词高亮其 bin。
- **可信度 vs 紧致度 / Shepard 迷你图**：x=高维距离 y=UMAP距离 + 对角线，偏离=失真；或 T/C 散点。

## 6) 逐步迭代 Step
1🟢 点云立住(自定义 ShaderMaterial+距离衰减) 2🟢 软光晕+加色 **3🟢 一维失真热力(Turbo, 2/98归一, 驱动色+大小+亮度) 4🟢 配色图例条** 5🟢 景深三件套(雾+vignette+热点豁免) 6🟡 hover tooltip 7🟡 近邻连线按 ratio 上色+阈值 8🟡 点击聚焦+Focus+Context 9🟡 自定义滑块面板 10🟡 直方图+Shepard图 11🔴 GPU color-id 拾取 12🔴 选择性 bloom(已做) 13🔴 CheckViz 2D 配色(专家模式默认关) 14🔴 屏幕空间 KDE 热区(HUD 可关)

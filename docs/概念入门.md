# 大模型研究入门 —— 为《潜空间银河 · UMAP 测谎仪》定制

> 读法：标 **★ = 必须懂、汇报要讲**；标 ○ = 背景了解即可，不用深究。
> 渲染部分你熟，本文不讲渲染，只讲"我们到底在研究什么"。
> 一句话先记住：**这个项目 = 给"看大模型内部的工具"测谎。**

---

## 0. 三分钟速览（先有个全局）

- 大模型（LLM）说白了是个**超大号"下一个词预测器"**：给它一段话，它猜下一个最可能的词，一个个接龙，就成了回答。
- 它能"猜得准"，是因为训练时把每个词、每句话都变成了**一串数字（向量）**，意思相近的东西数字也相近。这堆数字住在一个**几百维的空间**里——这就是模型的"脑内世界"，术语叫**潜空间 / 嵌入空间（latent / embedding space）**。
- 我们的项目**不碰**"怎么训练""输出对不对"，只干一件事：**把这个脑内世界画给人看，然后检查——我们用来画它的工具（降维算法 UMAP）到底有多少是真的、多少是它瞎编骗眼睛的。**

---

## 1. ★ 当前大模型研究分成哪几摊（你能在汇报里"定位"自己）

可以把整个领域通俗地分成 6 摊。你只要知道**我们在第 6 摊**，并能说出别的摊大概在干嘛即可：

| 研究方向 | 通俗说它在干嘛 | 备注 |
|---|---|---|
| ① 训练 / 后训练（对齐、RLHF） | 怎么把模型**造出来**、怎么**调教**得听话有礼貌 | 最烧钱烧卡，大厂主场 |
| ② 推理 / 智能体（Agent） | 怎么让它**会用工具、会多步思考、会自己干活** | 2024–2026 最热的应用方向之一 |
| ③ 评测（Eval） | 怎么**判断它到底行不行**、好在哪坏在哪 | **你同学的项目就在这摊** |
| ④ 效率 / 小型化 | 怎么让它**更快更省**（端侧、量化、蒸馏） | 让手机也能跑 |
| ⑤ 安全 / 对齐 | 怎么**别让它干坏事**、别被骗、别越狱 | 防御方向 |
| ⑥ **可解释性（Interpretability）** | **打开它的"脑子"，看里面到底怎么表示知识、怎么思考** | ← **我们在这摊** |

**一句"现状大势"**（汇报可用）：2023–2026 年，行业重心正从"**把模型做得更大**"，逐渐转向"**把模型用好、看懂、管住**"。其中"看懂模型内部"（可解释性）越来越受重视——因为模型越强、越被用在重要场合，"它内部到底在想什么、我们对它的理解靠不靠谱"就越关键。我们的项目正是对"理解工具本身"的一次较真。

> ○ 进阶名词（听过即可）：机制可解释性（mechanistic interpretability）、稀疏自编码器特征（SAE features）、表征几何（representation geometry）。这些都是"打开脑子看内部"这一摊里更细的招式，你不必深究。

## 2. ★ 核心概念 A：模型内部怎么"表示"意思 —— 向量与高维空间

**关键直觉：模型把"意思"变成了"坐标"。**

- 每个词（更准确说是每个 token）被模型变成一串数字，比如 GPT-2 里是 **768 个数字** = 768 维空间里的一个坐标点。
- **意思相近 → 坐标相近。** 比如 "猫""狗""兔子" 会挤在一块儿，"周一""周二""周三" 挤在另一块儿，"国王 - 男 + 女 ≈ 王后" 这种关系也藏在坐标里。
- 一段话进入模型后，会**一层层往上加工**（GPT-2 有 12 层）。每一层都有一套坐标，越往上"理解"越抽象。所以同一个词在不同层有不同坐标——这给了我们"看它逐层流动"的素材。

这堆坐标住的空间，就叫**潜空间 / 嵌入空间**。它是模型"理解世界"的方式，也是我们这个项目要可视化的对象。

> 类比：模型把整个语言世界，安排成了一座几百维的"星图"，每个词是一颗星，意思近的星挨着。问题是——**这座星图有几百维，人只能看 3 维。**

---

## 3. ★★ 核心概念 B：为什么要"降维"，以及降维会骗人（这是项目的心脏）

**问题**：768 维人脑无法想象，要给人看，必须"压扁"到 2 维或 3 维。这一步叫**降维（dimensionality reduction）**。

常见三种降维方法，性格很不一样：

| 方法 | 性格 | 优点 | 缺点 |
|---|---|---|---|
| **PCA**（主成分分析） | **老实人** | 线性、保留"全局大结构"、点之间远近**相对可信** | 画出来常常糊成一团、不好看 |
| **t-SNE** | **戏精** | 把局部聚类分得特别开、特别好看 | 聚类间的距离、全局布局**大多没意义**，参数一调结果就变 |
| **UMAP** | **戏精**（比 t-SNE 快） | 同上，聚类干净漂亮 | 同上，**好看是以扭曲真实关系为代价的** |

**这就是项目的核心论点（已用真实数据自测、并做了关键修正）**：UMAP 不是"处处撒谎"，而是**局部诚实、全局撒谎**：
- **局部**：你的真邻居，UMAP 基本都帮你留在身边（它就是为保近邻设计的）。
- **全局**：簇与簇之间的距离、整体布局、那些"分得清清楚楚"的空隙——**大多是 UMAP 编的**，不反映模型里真实的远近。

> **地图投影类比**（汇报金句来源）：把圆地球（3D）画成平面地图（2D），**任何画法都会骗你**——墨卡托投影里格陵兰看着比非洲还大，其实非洲大 14 倍。UMAP 就是这种"投影法"：邻居关系(局部)还算准，但大陆之间的相对大小和距离(全局)是假的。

**怎么量化（两个指标，真实 GPT-2 数据实测）：**

| 指标 | 测什么 | PCA | UMAP |
|---|---|---|---|
| **局部可信度** Trustworthiness | 低维里的邻居在高维是不是真邻居 | 0.616 | **0.716**（UMAP 局部更诚实）|
| **全局保真** | 高维距离 ↔ 低维距离 的秩相关(Spearman) | **0.345** | 0.176（UMAP 全局严重撒谎）|

我们的"测谎"用的是**全局失真**（`1 − 全局保真`，逐词算）：**用颜色烧在星系上**——全局诚实=冷色，被全局编造=烧红（红/亮黄=骗得越狠）。配色用感知均匀的 **Turbo / viridis**（不用会骗眼睛的彩虹色）。

## 4. ★ 我们站在哪、贡献是什么（汇报的"立意"）

- 我们**不是**检测模型训练得好不好，**也不是**检测它输出对不对。
- 我们检测的是：**把模型内部那座几百维星图画给人看时，所用的"投影工具"本身可不可信。**
- **独特角度（你"自己的想法"）**：别人用 UMAP 去看模型，我们**反过来给 UMAP 测谎**——把可视化工具当成研究对象去证伪，给出一个可证伪、且经真实数据验证的硬结论：

> **UMAP「局部诚实、全局撒谎」：它保住了你的真邻居(局部可信度 0.72>PCA 0.62)，却把簇间距离和整体布局编成了假象(全局保真仅 0.18<PCA 0.35)。那些"好看的分离"是全局的谎。**

这比"UMAP 就是骗子"更准确、更有研究味（先生会更买账）。它**既是炫酷演示、又是真研究**：屏幕上"星系烧成红色"+ Shepard 图"点散成一团"的视觉爆点，本身就是这条结论的呈现。

## 5. ★ 汇报可能被问到的概念，一句话答案（背下来就够）

| 老师可能问 | 你可以照说 |
|---|---|
| "embedding（嵌入）是什么？" | "就是模型把每个词变成的一串数字坐标，意思相近的词坐标也相近。" |
| "UMAP 是什么？" | "一种把几百维压到 2/3 维好让人看的降维方法，保局部近邻很强，但会扭曲全局距离。" |
| "你怎么知道它在骗人？" | "测两个指标：局部可信度(UMAP 0.72 比 PCA 0.62 高=保近邻、局部诚实)，但全局保真(高维↔低维距离的秩相关，UMAP 只有 0.18 比 PCA 0.35 低=全局撒谎)。Shepard 图上 UMAP 的点散成一团就是铁证。" |
| "那 UMAP 到底好不好？" | "看用途。要找邻居/聚类它很好；但你**不能**信它的簇间距离和整体布局——那是它为了好看编的。这正是大家常踩的坑。" |
| "这跟模型质量有关系吗？" | "没关系，我们查的是'可视化工具'准不准，不是模型本身好不好——这正是容易被忽视的盲点。" |
| "为什么用 GPT-2 这种老模型？" | "它小、开源、可复现，足够把'内部表示+降维失真'讲清楚；这个问题跟模型大小无关，小模型反而干净好讲。" |
| "能实时跑别的输入吗？" | "能，本地后端几秒就出；为讲解效果明显，我也准备了精选案例。" |

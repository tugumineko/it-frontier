# 沉浸式银河系渲染 · 逐步迭代计划

> 目标：让人"置身银河系当中"，同时旋臂/结构清晰可辨、不被 bloom 洗白。
> 约束：Three.js r160、纯前端、离线、零外部贴图、中端显卡 60fps。
> 来源：5 方向开源技法调研（threejs-journey galaxy generator、IQ domain warping、官方 selective bloom、Codrops 程序化云、Maxime Heckel/pegwars 体积 raymarch 等，链接见末尾）。

## 0. 构成层次（看得清 + 沉浸 的关键）

5 层叠加 + 1 个后处理选择器。**核心设计铁律：只有"星核/亮星"这一层做 HDR 发光并进入 bloom，其余全部钳在阈值之下** → 结构永不被洗白。

| 层 | 内容 | 技法 | 亮度预算 | 进 bloom? |
|---|---|---|---|---|
| L0 | 星云穹顶背景霞 | BackSide 球 + 域扭曲 fbm + 尘埃遮罩 | ×0.3 | 否 |
| L1 | 深空星野 | 双层 Points + 程序软圆点 + 微闪 | <0.5 | 否 |
| L2 | 银河星盘主体 | 100k+ Points 螺旋臂 + core→rim 渐变 + 差速自转 | 0.6~0.8 | 否 |
| L3 | 尘埃带 | NormalBlending Points（遮挡，非加亮）| 暗 | 否 |
| L4 | 星系核/亮星 | 少量 Points/Mesh，颜色 ×(2~4) → HDR>1 | >1.0 | **是** |
| PP | 选择性 Bloom | 双 composer + darkenNonBloomed + 末端 OutputPass | — | — |

- **结构清晰** = L2 高对比小亮点 + L3 暗尘埃带"负空间" + L0 多项式压暗中灰。
- **不洗白** = L0~L3 亮度 ≤ bloom 阈值；仅 L4 HDR>1 被 bloom 抠出。
- **沉浸感** = L0 穹顶包裹 + L1 视差 + 大气透视纵深 + L2 差速自转运动感。

## 1. 两个 bug 的根因

- **简陋 billboard 星云**：加性混合无上限，多片重叠累积 >1 全 clamp 成白，加性永远做不出暗带/对比；且单片只是糊版 alpha 圆斑，无 fbm 丝缕。→ 用 L0 域扭曲 fbm 穹顶 + L3 NormalBlending 尘埃带替代。
- **bloom 整屏过曝**：① threshold 0.2 太低（深空大量中等亮度像素都 >0.2 被抠进 bloom）；② UnrealBloomPass 加性变亮（[#14899](https://github.com/mrdoob/three.js/issues/14899)，strength=0 仍约 ×2），调小 strength 治不了本。→ Step1 抬阈值 + 末端 OutputPass + Reinhard；Step2 选择性 bloom 根治。

## 2. 逐步迭代（Step1→8，立竿见影→锦上添花）

| Step | 目标 | 复杂度 | 新 vendored |
|---|---|---|---|
| **1 ✅** | 止血：bloom 不洗白（threshold 0.8/strength 0.4/Reinhard/HalfFloat RT）+ 去掉简陋 billboard，星云改域扭曲 fbm 穹顶 | low | 无 |
| 2 | 选择性 Bloom（双 composer + darkenNonBloomed），只星核发光、结构清晰 | mid | 无 |
| 3 | 银河星盘主体：100k Points 程序化螺旋臂（Bruno Simon 公式，core→rim，差速自转）—— 观感最大跃迁 | low | 无 |
| 4 | 深空 starfield 双层（纵深/视差/星海密度）| low | 无 |
| 5 | 尘埃带：NormalBlending 遮挡（负空间，识别"银河"的最强线索）| mid | 无 |
| 6 | 大气透视：FogExp2 + ShaderMaterial fog chunk | low | 无 |
| 7 | 星云穹顶进阶：fbm 渲到离屏 RT 贴 BackSide 天球（无接缝、低频更新省钱）| mid | 1（GLSL noise）|
| 8 | 锦上添花：1~3 处体积 raymarch 星云核（"穿过去"的纵深）| high | 无 |

调试顺序：先把 **Step1+2 的 bloom** 调对（确认只有星核发光），再 **Step3 星盘**（旋臂清晰），其余按需叠加。

## 3. 安全默认参数

```js
// 渲染器 / Tone Mapping
renderer.toneMapping = THREE.ReinhardToneMapping;  toneMappingExposure = 1.0;
pixelRatio = min(devicePixelRatio, 2);

// 选择性 Bloom（唯一发光层=星核/亮星）
UnrealBloomPass: strength 1.2, radius 0.4, threshold 0.85;  bloomComposer RT = HalfFloatType
mix 片元 = base + vec4(1.0)*bloom;  末端 OutputPass 唯一 tone map
// 非选择性退路（当前 Step1）: strength 0.4 / radius 0.4 / threshold 0.8

// L2 银河星盘
count 120000(沉浸 200k+), radius 5(放大 50~200), branches 3, spin 1,
randomness 0.2, randomnessPower 3, uSize~20, Y 压扁 0.35,
insideColor #ffeec0, outsideColor #3a6bd8, vColor 亮度≤0.8, pow(strength,10)

// L1 starfield: 远景 40000(1~2px,亮度0.3~0.6) + 近景 3000(2~6px,少数8~12)
// L0 星云穹顶: RT512 HalfFloat, fbm 4oct/lacunarity2.02, 亮度×0.3, 尘埃遮罩最暗0.15
// L3 尘埃带: 30000, alpha 0.6, 色(0.04,0.025,0.02), NormalBlending(绝不加性)
// 雾: FogExp2(0x05060a, 0.02) 雾色=背景色; gl_PointSize clamp(.,1,12)

// 亮度预算铁律：L0~L3 ≤ 0.8；仅 L4 星核 ×(2~4) → HDR>1 → 唯一进 bloom
```

# 代码词法地图 · 看懂每个单词

## 开场：为什么做这个

编程初学者面对一段代码时，最常见的困惑不是「这段代码做什么」，而是「这个词是什么意思」。现有的 AI 代码解释工具大多以**整行或整段**为粒度给出笼统说明，跳过了最基础的一步：逐个单词告诉你它属于哪类、在这里起什么作用。

代码词法地图就是为了解决这个问题。它把一段代码拆成一个个 token，按编译器的分类方式分到四个区域，然后用 LLM 逐个解释每个标识符在当前上下文中的具体含义。

## 演示流程

### 第一步：粘贴代码，点击分析

把一段 Python 代码粘贴到左侧输入框，点「分析」。大约 0.1 秒后，中央出现一张 2D 词法地图，所有 token 被分到四个区域：关键字、标识符、运算符、字面量。左侧代码面板也同步高亮着色。

这一步完全由 **Pygments 词法分析器**完成，不依赖任何 LLM，速度快且分类结果与编译器一致。

### 第二步：点击任意词，看它在这里干嘛

点击地图上的圆点或代码中的高亮词，三个视图同步联动：地图圆点变亮、代码行高亮、右下角弹出语义卡片。

语义卡片的内容来源分为三级：

- **固定字典**（self、cls、def、return 等）：毫秒级返回，因为这些词在语言层面含义恒定，不需要每次问 LLM。
- **上下文快速提示**（def 后面的词标注「函数名」，class 后面标注「类名」，import 后面标注「模块」）：同样毫秒级，基于前后 token 关系推断。
- **LLM 深层解释**：对于没有固定含义的标识符（如用户自定义变量名），点击后异步请求 ecnu-max，约 1 秒返回上下文相关的一句话解释。

如果连续点击多个标识符，所有请求**并发执行**，不会排队等待。

### 第三步：用分级样例引导

左侧提供五个分级样例按钮（变量、条件、循环、函数、一词多义），从最简单的 `x = 10` 到刻意设计的变量名重叠代码。一键点击即填入并自动分析，降低初学者「不知道从哪下手」的门槛。

### 第四步：导出与复用

分析结果可以导出为 JSON 文件，也可以从文件导入。教学场景下，教师可以预先分析好代码片段并分发给学生。

## 核心概念速览

| 概念 | 说明 |
|---|---|
| 词法分析 | 编译器的第一步，把源代码字符流切成有意义的最小单元（token） |
| 四大词法类别 | 关键字（语言保留词）、标识符（用户或库定义的名称）、运算符（操作符号）、字面量（写死的值） |
| 上下文语义 | 同一个词在不同位置可能承担不同角色，如 `data` 可以是模块别名、函数名、类名 |
| 分级解释 | 固定字典秒出、上下文提示先行、LLM 深层解释到了替换 |

## 收尾

这个工具把编译原理中「词法分析」这一步从黑箱变成了可交互的可视化体验，再用 LLM 补上编译器不提供的「这个词在这里干嘛」的语义信息。两者各司其职：词法分类交给确定性的 lexer（快、准、免费），语义解释交给 LLM（灵活、上下文相关）。

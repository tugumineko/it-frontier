# 代码词法地图 · 看懂每个单词 (Code Lexical Map)

> 面向**编程初学者**：粘贴一段代码 → 用编译器词法分析切成 token、分到「**关键字 / 标识符 / 运算符 / 字面量**」四区的 2D 地图 → agent（ecnu-max）逐个词讲「它是什么、在这里干嘛」。代码 ↔ 地图 ↔ 语义卡点选联动。

信息技术前沿创新课程项目。**词法身份**由 Pygments 词法分析器给（确定、快、多语言），**上下文语义**由 ECNU `ecnu-max` 用大白话解释。前端 SVG 2D + 星空背景，离线优先。

---

## 快速开始

1. 复制 `server/secrets.example.json` 为 `server/secrets.json`，填 ECNU 的 `base_url`/`api_key`（无凭据也能看词法分类，配上后才有逐词解释）。
2. 装依赖：`pip install -r server/requirements.txt`（flask + pygments）。
3. 启动：`python server/app.py`，浏览器开 http://127.0.0.1:5000。

## 怎么用

1. 选语言、粘贴代码，或点**样例**（变量 / 条件 / 循环 / 函数 / 一词多义）→「分析」。
2. 看 **2D 词法地图**：token 按 4 类落进各自分区，点的大小 = 出现频率，连线 = 代码顺序的「词法轨迹」。
3. **点代码里的词、或地图上的点**：语义卡显示「这是什么词类 + 在这里干嘛」，代码 / 地图 / 卡片三处联动高亮。
4. 导出 / 导入词法地图 JSON（离线可回放）。

## 架构

- **后端** `server/app.py`：`/api/analyze` —— Pygments 词法切 token + 4 类 + 频率权重；一次调 ecnu-max 给关键字/标识符逐词大白话语义。词法身份交给确定性 lexer，LLM 只补「在这里干嘛」。
- **前端** `web/`：SVG 2D 词法地图 + 代码面板 + 语义卡，点选共享联动。

## 定位

不是给编译原理学生的词法调试器，也不是给开发者的 AST 工具，而是帮**零基础初学者**看懂「一段陌生代码由哪几类词构成、每个词在这里是什么意思」——把编译器的词法类别骨架，和 LLM 的逐词上下文解释，在 token 粒度对齐讲出来。

> 课程 demo，重点在教学与演示，非生产级。
